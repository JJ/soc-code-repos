Finding the right level is key

But is key for the correct description of the system and being able to take some decisions on it. It's actually the same system, as described in Gershenson's thesis. It's also got to do with the way we are observing changes: using commits. If we used days, or weeks, instead of this natural unit, the result might be much different, and self-organization might not take place at all.

(How|What|Who) * self-organization

What/why is critical

This part is clear, and it's also related to how creativity works: in avalanches and bursts. One change leads to many others, and a small change in a situation might lead to the rewriting of a whole chapter. That happens in software development too, and it's not exclusive of using a code repository. Writing in longhand in notebooks also will have a very similar effect, only it would be more difficult to measure.
